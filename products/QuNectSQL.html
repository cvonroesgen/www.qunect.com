QuNect SQL

Build, Check Syntax and Execute SQL Statements Against Quickbase

Please install the prerequisite QuNect ODBC for QuickBase first!

QuNect SQL builds SELECT, INSERT, UPDATE, DELETE, CREATE TABLE, ALTER TABLE and DROP TABLE statements with the press of a button. Then you can edit the resulting statement or execute it as is with the press of a button. You can also write your own SQL statements and check their syntax before executing them. QuNect SQL is available for a 30 day free trial. After the trial expires a license for QuNect ODBC for QuickBase is required. Licensees of QuNect ODBC for QuickBase can use QuNect SQL for free. QuNect SQL requires MS Windows and .NET Framework 4.5. QuNect SQL also has command line capability. You need to supply two command line arguments. The first is the DSN that you want to use and the second is a file including its full path. The file needs to contain a single SQL statement that you want executed. For example: 

		C:\QuNect\automation>QuNectSQL.exe "QuickBase via QuNect 64 bit" "C:\QuNect\QuNectSQL\uploadfile.sql" >> log.txt

The above command line will append any errors that occur into a text file called "log.txt". If you don't want to log the errors then use the example below: 

		C:\QuNect\automation>QuNectSQL.exe "QuickBase via QuNect 64 bit" "C:\QuNect\QuNectSQL\uploadfile.sql"

The command line below will write the output of a SELECT statement to the file specified by the third command line argument: 

		C:\QuNect\automation>QuNectSQL.exe "QuickBase via QuNect 64 bit" "C:\QuNect\QuNectSQL\select.sql" "C:\QuNect\QuNectSQL\select.csv"
