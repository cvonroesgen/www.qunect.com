Application Note for QuNect ODBC for QuickBase

Minimizing Integration Reads with QuNect ODBC for QuickBase

  • Use a user token in your DSN or connection string instead of a password to eliminate one integration read every time you establish an ODBC connection.
  • Use INSERTs instead of UPDATEs.
  • Use table reports instead of a WHERE clause in SELECT, UPDATE and DELETE statements.
  • Use OpenQuery instead of Four Part Names
  • Use UPDATEOQ, DELETEOQ with OpenQuery when possible.
  • Use the DOQUERYCHUNKSIZE connection string parameter to increase the number of records returned per API_DoQuery API call.
  • Use QuNect ODBC for QuickBase API and SQL logging temporarily to look for API_DoQuery and API_GenResultsTable API calls and the SQL statements that cause them.
